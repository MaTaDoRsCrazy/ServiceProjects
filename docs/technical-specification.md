# Техническое задание

## Наименование

Corporate ServiceDesk - информационная система учета заявок и оборудования организации.

## Основание для разработки

Система разрабатывается как учебно-практический проект по специальности 09.02.07
«Информационные системы и программирование».

## Цель проекта

Автоматизировать обработку обращений сотрудников в ИТ-службу и вести единый реестр
оборудования организации.

## Пользователи

| Роль | Возможности |
| --- | --- |
| Администратор | Управление заявками, оборудованием, просмотр отчетов |
| Техник | Просмотр заявок, смена статуса, добавление комментариев |
| Сотрудник | Создание заявок, просмотр собственных заявок, комментарии |

## Функциональные требования

- Авторизация пользователей по логину и паролю.
- Выдача JWT-токена для доступа к REST API.
- Создание заявки с темой, описанием, приоритетом и привязкой к оборудованию.
- Просмотр списка заявок с фильтрацией по статусу.
- Изменение статуса заявки ответственным сотрудником.
- Добавление комментариев к заявке.
- Ведение справочника оборудования.
- Просмотр отчетной панели по заявкам и оборудованию.
- Доступ к системе через web, desktop и Android-клиент.

## Нефункциональные требования

- Backend должен быть реализован на Java и Spring Boot.
- Хранение данных должно выполняться в PostgreSQL.
- Доступ к API должен быть защищен Spring Security и JWT.
- Проект должен запускаться через Docker Compose.
- Код должен содержать JUnit-тесты для ключевой логики.
- Документация должна описывать API, БД, тестирование и сценарии работы.

## Технологический стек

| Часть | Технологии |
| --- | --- |
| Backend | Java 17, Spring Boot, Spring Security, JPA/Hibernate, Maven |
| База данных | PostgreSQL |
| Web | HTML, CSS, JavaScript, Nginx |
| Desktop | Kotlin/JVM, Swing |
| Mobile | Android Kotlin |
| DevOps | Docker, Docker Compose |
| Тестирование | JUnit 5, Mockito |

## Критерии приемки

- Пользователь может войти в систему.
- После входа доступны заявки и оборудование.
- Новая заявка сохраняется в базе данных.
- Статус заявки изменяется через API и отображается в интерфейсах.
- Отчетная панель показывает агрегированные показатели.
- Проект запускается командой `docker compose up --build`.
