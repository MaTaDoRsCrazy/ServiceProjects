# План тестирования

## Цель

Проверить корректность авторизации, обработки заявок, учета оборудования и отчетности.

## Виды тестирования

| Вид | Что проверяется |
| --- | --- |
| Модульное | JWT, агрегирование отчетов |
| Интеграционное | REST API, доступ к БД, авторизация |
| Функциональное | Сценарии пользователя, техника и администратора |
| UI-тестирование | Web, desktop и Android-клиент |
| Нагрузочное базовое | Открытие списков заявок и оборудования |

## Тестовые сценарии

| ID | Сценарий | Ожидаемый результат |
| --- | --- | --- |
| T-01 | Вход с корректным логином и паролем | Возвращается JWT-токен |
| T-02 | Вход с неверным паролем | Возвращается ошибка авторизации |
| T-03 | Создание заявки сотрудником | Заявка получает статус `NEW` |
| T-04 | Смена статуса техником | Статус сохраняется в БД |
| T-05 | Добавление комментария | Комментарий отображается в списке |
| T-06 | Просмотр оборудования | Возвращается список карточек |
| T-07 | Удаление оборудования сотрудником | Доступ запрещен |
| T-08 | Просмотр отчета администратором | Возвращаются агрегированные показатели |

## Автоматические тесты

В backend добавлены JUnit-тесты:

- `JwtServiceTest` проверяет генерацию и валидацию JWT.
- `ReportServiceTest` проверяет расчет сводных показателей.

Запуск:

```powershell
cd backend
mvn test
```

## Ручная проверка через curl

```powershell
$body = @{ username = "admin"; password = "admin123" } | ConvertTo-Json
$auth = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/auth/login" -Body $body -ContentType "application/json"
$headers = @{ Authorization = "Bearer $($auth.token)" }
Invoke-RestMethod -Uri "http://localhost:8080/api/tickets" -Headers $headers
```
